DOCUMENTATION

Hybrid Truncated NFW External Potential
=======================================

Overview
--------

This mode combines two regimes around a user-defined cutoff radius :math:`R_{cut}`:

1. Inner region (:math:`r \le R_{cut}`):
   - No external NFW acceleration is applied.
   - Particles evolve with normal SWIFT self-gravity.

2. Outer region (:math:`r > R_{cut}`):
   - External NFW acceleration is applied.
   - Dark matter particles can be made massless for self-gravity (DM-only mask).

This backend is selected with:

- configure option: --with-ext-potential=truncated-nfw
- macro path in code: configure.ac, potential.h

Physical interpretation
-----------------------

This setup is designed for hybrid experiments where the inner halo is evolved self-consistently, while the outer region is represented by an imposed NFW field and (optionally) DM particles outside :math:`R_{cut}` do not source self-gravity.

Important: the center used for both the external potential and the DM mask is the same center read from NFWPotential:position. It is not auto-detected from the halo particle distribution.

How the center is defined
-------------------------

The potential center is read from:

- NFWPotential:position in potential.h
- NFWPotential:useabspos in potential.h

Interpretation:

1. useabspos = 1:
   - position is absolute simulation coordinates.

2. useabspos = 0:
   - position is interpreted relative to box center and shifted by half-box size in each axis.

The same center is propagated to DM masking globals in potential.c.

How the cutoff is defined
-------------------------

The cutoff is user-defined in the parameter file:

- required: NFWPotential:cutoff_radius in potential.h
- optional: NFWPotential:dm_mask_outside_cutoff (default 1) in potential.h

Region classification uses:

- :math:`r^2 > R_{cut}^2` as outer region in potential.h

Behavior implemented
--------------------

External potential behavior:

- Inner region:
  - no external acceleration in potential.h
  - no external potential energy term in potential.h
  - no extra external timestep constraint in potential.h

- Outer region:
  - standard NFW external acceleration and potential term.

DM-only mass masking behavior:

- Applied only to particle type swift_type_dark_matter in gravity.h
- Effective mass is set to zero outside cutoff when enabled.
- Effective mass is used in:
  - local and foreign packed masses gravity.h
  - gravity caches gravity_cache.h
  - multipole construction multipole.h
  - no-cache pair loops runner_doiact_grav.c

Build and run
-------------

1. Configure SWIFT with this backend:

   ./configure --with-ext-potential=truncated-nfw [other options]

2. Build as usual.

3. Provide an NFWPotential block in the YAML parameter file.

Parameter file example
----------------------

Use the NFWPotential section (same family of keys as NFW, plus cutoff keys):

::

    NFWPotential:
      useabspos: 0
      position: [0.0, 0.0, 0.0]
      timestep_mult: 0.01
      concentration: 9.0
      M_200: 137.0
      h: 0.704
      epsilon: 0.2
      bulgefraction: 0.0
      diskfraction: 0.04

      cutoff_radius: 30.0
      dm_mask_outside_cutoff: 1

Notes:

- cutoff_radius is required.
- dm_mask_outside_cutoff is optional and defaults to 1.
- M_200 key name follows the implementation key exactly.

Practical guidance for choosing center and cutoff
-------------------------------------------------

1. Center:
   - Set position to the actual dynamical center of the DM halo (or intended fixed center of the external field).
   - If the halo drifts and you keep a fixed center, the boundary is evaluated around that fixed point.

2. Cutoff radius:
   - Choose based on where you want to switch from self-gravitating halo to externally imposed NFW regime.
   - Transition is a hard step at :math:`R_{cut}`.

3. Scope:
   - DM masking is DM-only by design.
   - Gas, stars, sinks, and black holes are not mass-masked by this feature.

Diagnostics
-----------

At startup, the backend prints:

- truncated NFW activation
- center, scale radius, halo properties
- cutoff radius and DM masking flag

See logging in potential.h.

Known limitations
-----------------

1. The cutoff center and external potential center are currently tied to the same position.
2. The transition at :math:`R_{cut}` is discontinuous (hard boundary).
3. Center is user-specified; there is no automatic halo-center finder in this backend.

validation checks
---------------------------

1. Place tracer particles just inside and just outside :math:`R_{cut}` and verify:
   - inner: no external NFW acceleration
   - outer: non-zero external NFW acceleration

2. For DM particles outside :math:`R_{cut}` with dm_mask_outside_cutoff=1, verify they do not source self-gravity.

3. Compare single-rank and MPI runs for consistency of the masked regime.
